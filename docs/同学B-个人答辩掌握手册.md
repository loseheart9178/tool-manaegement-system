# 同学 B 个人答辩掌握手册

适用分工：借还管理、借还记录文件读写、库存统计、低库存预警、借用排行榜。

## 1. 你要能用 1 分钟讲清楚自己做了什么

我主要负责工具借还管理和统计分析模块。借还管理围绕 `BorrowRecord` 结构体展开，实现了借用工具、归还工具、查看借还记录、记录编号自动生成和借还记录文件保存加载。统计分析部分基于工具数组和借还记录数组，实现库存总量、在库数量、已借数量、总价值统计，低库存预警，以及借用次数排行榜。

我的模块重点是保证借用和归还时库存同步更新，避免超库存借用、损坏工具借用和重复归还。

## 3. 核心数据结构怎么讲

借还记录用 `BorrowRecord` 表示：

```c
typedef struct
{
    int record_id;
    char tool_id[20];
    char borrower_name[BORROWER_NAME_LEN];
    char borrow_date[DATE_LEN];
    char due_date[DATE_LEN];
    int quantity;
    char status[BORROW_STATUS_LEN];
} BorrowRecord;
```

答辩讲法：

这个结构体对应任务书里的借还记录表。`record_id` 是借还记录编号，自动生成并保持唯一；`tool_id` 用来关联工具信息表；`borrower_name`、`borrow_date`、`due_date` 记录借用人和时间；`quantity` 是借用数量；`status` 表示当前记录是“已借”还是“已还”。

老师如果问：为什么借还记录里只存 `tool_id`，不直接存完整工具信息？

回答：

因为借还记录和工具信息是关联关系。工具的名称、类型、库存等基础信息已经保存在 `Tool` 数组里，借还记录只需要保存工具编号作为关联字段。这样避免重复存储，也方便归还时通过工具编号找到对应工具并更新库存。

## 4. 借还管理模块怎么讲

### 4.1 生成记录编号

函数：`generate_record_id`

逻辑：

- 如果没有记录，返回 1。
- 如果已有记录，遍历所有记录找到最大 `record_id`。
- 返回最大编号加 1。

老师可能问：为什么不直接用 `record_count + 1`？

回答：

如果以后有删除记录或导入历史记录的情况，`record_count + 1` 可能和已有编号重复。用最大编号加 1 更稳妥。

### 4.2 查找借还记录

函数：`find_record_index_by_id`

作用：根据记录编号查找数组下标，找不到返回 `-1`。

它主要在归还功能里使用，用来定位要归还的那条记录。

### 4.3 借用工具

函数：`borrow_tool`

你要背下来的流程：

1. 判断借还记录数组是否已满。
2. 输入工具编号。
3. 调用 `find_tool_index_by_id` 判断工具是否存在。
4. 如果工具状态是“损坏”，禁止借用。
5. 输入借用数量。
6. 校验数量必须大于 0，且不能超过当前库存。
7. 生成新的借还记录编号。
8. 输入借用人、借用日期、应还日期。
9. 记录状态设为“已借”。
10. 更新工具库存：
    - `stock -= quantity`
    - `borrowed_count += quantity`
11. 把新记录加入 `records` 数组，`record_count` 加 1。

老师可能问：借用成功后为什么要同时改两个字段？

回答：

因为 `stock` 表示当前仓库里还能借的数量，`borrowed_count` 表示已经借出的数量。借出以后，在库数量减少，已借数量增加，这两个字段必须同步更新，库存统计才准确。

### 4.4 归还工具

函数：`return_tool`

你要背下来的流程：

1. 输入借还记录编号。
2. 查找对应借还记录。
3. 如果记录不存在，返回失败。
4. 如果状态已经是“已还”，禁止重复归还。
5. 根据记录中的 `tool_id` 找到对应工具。
6. 恢复库存：
    - `stock += quantity`
    - `borrowed_count -= quantity`
7. 把记录状态改成“已还”。

老师可能问：为什么归还时根据记录编号，而不是直接输入工具编号？

回答：

因为同一个工具可能被不同人多次借用，单靠工具编号无法确定是哪一次借用。记录编号是唯一的，可以准确定位具体借用记录，避免归还错记录。

### 4.5 查看借还记录

函数：`show_all_records`

作用：以表格显示所有借还记录，包括记录编号、工具编号、借用人、借用日期、应还日期、数量、状态。

这个功能主要用于历史追溯，也方便老师检查借用和归还后状态是否变化。

## 5. 借还记录文件读写怎么讲

文件：`src/borrow.c`

### 5.1 保存记录

函数：`save_records`

保存方式：

1. 先写入 `record_count`。
2. 再循环写入每条 `BorrowRecord`。

### 5.2 加载记录

函数：`load_records`

加载方式：

1. 打开二进制文件。
2. 读取记录数量。
3. 判断数量是否合法。
4. 最多读取 `max_records` 条，防止数组越界。
5. 设置 `*record_count`。

老师可能问：为什么工具记录和借还记录要分两个文件？

回答：

因为它们是两类不同的数据。工具信息相当于基础资料，借还记录相当于业务流水。分开保存结构更清晰，也方便分别加载和维护。

## 6. 统计分析模块怎么讲

### 6.1 库存整体统计

函数：`show_inventory_summary`

统计内容：

- 工具种类数：`tool_count`
- 库存总数量：`stock + borrowed_count`
- 当前在库数量：`stock`
- 已借出数量：`borrowed_count`
- 库存总价值：`(stock + borrowed_count) * price`

老师可能问：为什么总价值用 `stock + borrowed_count`，不是只用 `stock`？

回答：

因为已借出的工具仍然属于工厂资产，只是暂时不在仓库里。如果统计资产总价值，应该包括在库和已借出的全部工具。

### 6.2 低库存预警

函数：`show_low_stock_tools`

逻辑：

- 用户输入安全库存阈值。
- 遍历所有工具。
- 如果 `tools[i].stock < threshold`，就输出该工具。

答辩讲法：

低库存预警是一个实用功能，帮助仓库管理员发现当前可借库存不足的工具。这里比较的是当前在库数量 `stock`，因为它代表还能立即借出的数量。

### 6.3 借用次数排行榜

函数：`show_borrow_ranking`

实现步骤：

1. 创建一个临时排行榜数组 `RankEntry`。
2. 每个工具初始化一个计数，初始为 0。
3. 遍历所有借还记录，根据 `tool_id` 给对应工具计数加 1。
4. 用 `qsort` 按借用次数降序排序。
5. 输出借用次数大于 0 的工具。
6. 最后 `free(ranking)` 释放动态内存。

老师可能问：为什么这里用了 `malloc`？

回答：

排行榜的大小取决于当前工具数量 `tool_count`，运行时才知道，所以用 `malloc` 动态申请。使用完后调用 `free` 释放，避免内存泄漏。

老师可能问：`qsort` 是什么？

回答：

`qsort` 是 C 标准库提供的通用排序函数。这里传入排行榜数组、元素数量、元素大小和比较函数 `rank_cmp`，让它按借用次数从高到低排序。

## 8. 老师追问时的标准回答

### 问：你这个借还功能怎么保证库存不会变成负数？

答：

借用前会判断借用数量是否大于当前库存。如果 `quantity > tools[tool_idx].stock`，就直接返回失败，不会生成借还记录，也不会扣减库存。因此正常借用流程不会让库存变成负数。

### 问：如果重复归还同一条记录会怎样？

答：

归还前会检查记录状态。如果该记录的 `status` 已经是“已还”，程序会提示不能重复归还并返回失败。这样可以避免库存被重复加回去。

### 问：排行榜统计的是借用次数还是借用数量？

答：

当前统计的是借用次数，也就是某个工具在借还记录中出现了多少次。如果要统计借用数量，也可以把加 1 改成加 `records[i].quantity`。

### 问：你和同学 A 的模块怎么衔接？

答：

我的模块通过 `find_tool_index_by_id` 查找工具。借用和归还时，我不会自己维护另一套工具数据，而是直接修改主程序传进来的 `Tool tools[]` 数组中的 `stock` 和 `borrowed_count`。这样工具管理、借还管理和统计分析看到的是同一份数据。

## 9. 你答辩前必须能手写的代码片段

### 9.1 借用时更新库存

```c
tools[tool_idx].stock -= quantity;

```

### 9.2 归还时恢复库存

```c
tools[tool_idx].stock += records[rec_idx].quantity;
tools[tool_idx].borrowed_count -= records[rec_idx].quantity;
```

### 9.3 记录编号生成

```c
int max_id = records[0].record_id;
for (int i = 1; i < record_count; i++)
{
    if (records[i].record_id > max_id)
        max_id = records[i].record_id;
}
return max_id + 1;
```

### 9.4 排行榜比较函数

```c
static int rank_cmp(const void *a, const void *b)
{
    return ((const RankEntry *)b)->count - ((const RankEntry *)a)->count;
}
```
