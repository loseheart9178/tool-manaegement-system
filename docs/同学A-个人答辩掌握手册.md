# 同学 A 个人答辩掌握手册

适用分工：工具基础信息管理、文件存储、登录菜单、系统整合。

## 1. 你要能用 1 分钟讲清楚自己做了什么

我主要负责系统的基础框架和工具信息管理部分。程序从 `main.c` 进入 `run_system`，先进行登录验证，然后从文件中加载工具数据和借还记录，之后进入菜单循环。我的模块主要围绕 `Tool` 结构体展开，实现了工具的显示、添加、删除、修改和查询，同时负责工具数据的保存与加载，保证程序退出后数据不会丢失。

我这部分的重点不是单个输入输出，而是保证工具编号唯一、库存和单价合法、删除时防止误操作，并且让同伴的借还模块可以通过工具编号找到对应工具。

## 3. 核心数据结构怎么讲

工具信息用结构体 `Tool` 表示：

```c
typedef struct
{
    char id[TOOL_ID_LEN];
    char name[TOOL_NAME_LEN];
    char type[TOOL_TYPE_LEN];
    int stock;
    char status[TOOL_STATUS_LEN];
    int borrowed_count;
    char location[TOOL_LOCATION_LEN];
    double price;
} Tool;
```

答辩讲法：

这个结构体对应任务书里的工具信息表。`id` 是工具编号，作为主键使用，不能重复；`stock` 表示当前仓库里还能借出的数量；`borrowed_count` 表示已经借出去的数量；`status` 用来区分正常和损坏，损坏工具不能被借用；`price` 用于库存价值统计。

老师如果问“为什么不用很多单独数组”，回答：

因为一个工具有多个相关字段，如果用多个数组，比如编号数组、名称数组、库存数组，就要靠下标强行对应，容易出错。结构体能把一条工具记录封装成一个整体，更符合数据表的设计。

## 4. 工具管理模块怎么讲

### 4.1 查找工具

函数：`find_tool_index_by_id`

文件：`src/tool.c`

作用：根据工具编号在工具数组中查找下标，找不到返回 `-1`。

答辩讲法：

这是工具模块里最基础的公共函数。添加工具时用它检查编号是否重复；删除和修改时用它定位工具；借还模块也调用它判断工具是否存在。所以它相当于工具编号到数组下标的转换入口。

### 4.2 显示工具

函数：`show_all_tools`

作用：用表格形式输出所有工具。

要点：

- 如果 `tool_count == 0`，显示暂无工具记录。
- 否则循环输出每个 `Tool` 的字段。
- 使用格式化输出保证列表整齐。

### 4.3 添加工具

函数：`add_tool`

你要背下来的流程：

1. 判断工具数组是否已满。
2. 输入工具编号。
3. 调用 `find_tool_index_by_id` 检查编号是否重复。
4. 输入名称、类型、库存、状态、位置、单价。
5. 检查库存不能为负，状态只能是“正常”或“损坏”，单价不能为负。
6. 新工具的 `borrowed_count` 初始为 0。
7. 把新工具放到 `tools[*tool_count]`，然后 `(*tool_count)++`。

老师可能问：为什么 `tool_count` 要传指针？

回答：

因为添加成功后工具数量要改变。如果只传普通整数，函数内部改变的是副本，主程序里的数量不会变。传 `int *tool_count` 后，可以通过 `(*tool_count)++` 修改主程序中的真实数量。

### 4.4 删除工具

函数：`delete_tool`

你要背下来的流程：

1. 根据编号查找工具下标。
2. 找不到就提示失败。
3. 如果 `borrowed_count > 0`，说明还有工具在外面，禁止删除。
4. 删除前要求用户输入 `y/n` 二次确认。
5. 删除时把后面的数组元素整体前移。
6. `(*tool_count)--`。

老师可能问：为什么删除时要前移数组？

回答：

因为工具是用顺序数组保存的，删除中间元素后，如果不前移，中间会出现空洞，后续遍历显示和查找会不方便。前移后数组仍然保持连续。

### 4.5 修改工具

函数：`update_tool`

要点：

- 工具编号不能修改，因为编号是主键。
- 使用 `fgets` 读取新值。
- 用户直接回车时保留原值。
- 库存和单价要做非负校验。
- 状态只能是“正常”或“损坏”。

老师可能问：为什么修改功能里用 `fgets`，而添加里多用 `scanf`？

回答：

修改功能允许用户直接回车保留原值，所以需要读取整行输入并判断第一位是不是换行符。`scanf` 会跳过空白字符，不适合判断用户是否直接回车。

### 4.6 查询工具

函数：`search_tools`

支持三种：

- 按编号精准查询：`strcmp(tools[i].id, keyword) == 0`
- 按名称模糊查询：`strstr(tools[i].name, keyword) != NULL`
- 按类型模糊查询：`strstr(tools[i].type, keyword) != NULL`

老师可能问：什么是模糊查询？

回答：

模糊查询就是不要求完全相等，只要名称或类型中包含输入的关键字就算匹配。代码里使用 `strstr` 判断一个字符串是否包含另一个字符串。

## 5. 文件保存和加载怎么讲

文件：`src/storage.c`

### 5.1 保存工具

函数：`save_tools`

保存格式：

1. 先写入一个整数 `tool_count`。
2. 再逐条写入 `Tool` 结构体。

答辩讲法：

我使用二进制文件保存数据。保存时先写记录数量，再写具体记录。这样加载时先知道应该读取多少条，再循环读取结构体。

### 5.2 加载工具

函数：`load_tools`

流程：

1. 用 `fopen(filename, "rb")` 打开文件。
2. 如果文件不存在，返回 0，系统从空数据开始。
3. 读取记录数量。
4. 判断数量是否合法。
5. 最多读取 `max_tools` 条，避免数组越界。
6. 设置 `*tool_count = load_count`。

老师可能问：为什么要传 `max_tools`？

回答：

为了防止文件中的记录数超过数组容量。如果不限制，读取时可能写出数组范围，造成程序崩溃或数据破坏。

## 6. 菜单和主流程怎么讲

```text
main
  -> run_system
      -> login
      -> load_tools / load_records
      -> while 菜单循环
```

你可以这样讲：

## 8. 老师追问时的标准回答

### 问：你的模块和同伴模块怎么交互？

答：

我们通过共享的 `Tool tools[]` 数组和 `find_tool_index_by_id` 函数交互。我的模块维护工具基础信息，同伴的借还模块在借用或归还时，会根据工具编号找到工具下标，然后更新 `stock` 和 `borrowed_count`。这样两边不会各自维护一套库存，避免数据不一致。

### 问：你觉得这个系统最关键的数据一致性是什么？

答：

最关键的是库存一致性。对于每个工具，`stock` 表示当前在库数量，`borrowed_count` 表示已借数量。借用时库存减少、已借增加；归还时库存增加、已借减少。删除工具时如果已借数量大于 0 就不能删除，防止借还记录失去对应工具。

### 问：这个项目还有什么可以改进？

答：

目前使用的是固定大小数组，优点是简单稳定，符合课程要求；如果继续改进，可以改成链表或动态数组，支持更多数据。文件存储现在是二进制结构体直接写入，后续也可以改成文本或 CSV，方便人工查看和导入导出。

## 9. 你答辩前必须能手写的代码片段

### 9.1 按编号查找工具

```c
int find_tool_index_by_id(Tool tools[], int tool_count, const char tool_id[])
{
    for (int i = 0; i < tool_count; i++)
    {
        if (strcmp(tools[i].id, tool_id) == 0)
            return i;
    }
    return -1;
}
```

### 9.2 删除数组元素时前移

```c
for (int i = idx; i < *tool_count - 1; i++)
{
    tools[i] = tools[i + 1];
}
(*tool_count)--;
```

### 9.3 保存数据的基本模式

```c
fwrite(&tool_count, sizeof(int), 1, fp);
for (int i = 0; i < tool_count; i++)
{
    fwrite(&tools[i], sizeof(Tool), 1, fp);
}
```
