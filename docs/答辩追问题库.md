# 答辩追问题库

使用方法：两个人轮流抽问。回答时必须说出“在哪个文件、哪个函数、为什么这样做”。

## 1. 基础理解题

### Q1：这个项目解决什么问题？

参考回答：

解决工厂仓库工具管理问题，包括工具基础信息维护、工具借用归还、借还记录查询、库存统计、低库存预警和数据持久化保存。

### Q2：系统从哪里开始运行？

参考回答：

从 `src/main.c` 的 `main` 函数开始，`main` 调用 `run_system()`，具体登录、加载数据、菜单循环和退出保存都在 `src/menu.c` 中完成。

### Q3：系统用了几个主要结构体？

参考回答：

两个。`Tool` 表示工具信息，定义在 `include/tool.h`；`BorrowRecord` 表示借还记录，定义在 `include/borrow.h`。

### Q4：为什么要分模块写？

参考回答：

分模块能让职责更清晰。工具管理放在 `tool.c`，借还管理放在 `borrow.c`，统计分析放在 `statistics.c`，文件保存放在 `storage.c`。这样两个人可以并行开发，也方便调试和答辩说明。

## 2. 同学 A 高频问题

### Q1：添加工具时做了哪些校验？

参考回答：

在 `add_tool` 中，先检查数组是否已满，再检查工具编号不能为空且不能重复；库存必须大于等于 0；状态只能是“正常”或“损坏”；单价必须大于等于 0。

### Q2：工具编号唯一性怎么实现？

参考回答：

通过 `find_tool_index_by_id` 实现。它遍历工具数组，用 `strcmp` 比较每个工具的 `id` 和输入编号，如果找到相同编号就返回下标，添加时发现返回值不是 `-1` 就拒绝添加。

### Q3：为什么删除已借出的工具会被禁止？

参考回答：

因为如果工具已经借出，借还记录仍然引用这个工具编号。此时删除工具会导致借还记录找不到对应工具，也会造成库存数据不一致。所以 `delete_tool` 中判断 `borrowed_count > 0` 时禁止删除。

### Q4：修改工具时为什么直接回车可以保留原值？

参考回答：

因为修改功能使用 `fgets` 读取一整行，如果读到的第一位是 `\n`，说明用户直接回车，就不修改该字段。

### Q5：文件保存为什么使用二进制？

参考回答：

二进制保存可以直接写入结构体，代码简单，读取时也能直接恢复结构体数组。课程实践里数据格式固定，用二进制文件比较方便。

## 3. 同学 B 高频问题

### Q1：借用工具时做了哪些校验？

参考回答：

在 `borrow_tool` 中，先检查借还记录数组是否已满；再检查工具编号是否存在；如果工具状态是“损坏”则不能借；借用数量必须大于 0 且不能超过当前库存。

### Q2：借用成功后数据怎么变化？

参考回答：

新增一条 `BorrowRecord`，状态为“已借”；同时修改对应工具的库存：`stock` 减去借用数量，`borrowed_count` 加上借用数量。

### Q3：归还时为什么要检查状态？

参考回答：

为了防止重复归还。如果一条记录已经是“已还”，再次归还会导致库存重复增加，所以 `return_tool` 会先判断 `status` 是否为“已还”。

### Q4：为什么借还记录编号要自动生成？

参考回答：

因为记录编号是唯一标识，用于归还时准确定位某一次借用。自动生成能避免用户手动输入重复编号。

### Q5：排行榜的实现思路是什么？

参考回答：

在 `show_borrow_ranking` 中，先为每种工具创建一个计数项，然后遍历所有借还记录，记录中出现一次对应工具编号就给该工具计数加 1。最后使用 `qsort` 按次数降序排序并输出。

## 4. 代码细节题

### Q1：`strcmp` 和 `strstr` 分别用在哪里？

参考回答：

`strcmp` 用于精准比较，比如工具编号是否相同、状态是否等于“正常”或“损坏”。`strstr` 用于模糊查询，比如名称或类型中是否包含关键字。

### Q2：为什么有些参数是数组，有些参数是指针？

参考回答：

数组用于传递工具列表或记录列表。数量如果需要在函数内部改变，就传指针，比如 `int *tool_count` 和 `int *record_count`；如果只是读取数量，就传普通整数。

### Q3：`fwrite` 和 `fread` 是怎么配合的？

参考回答：

保存时先用 `fwrite` 写入记录数量，再写入每条结构体。加载时按同样顺序先用 `fread` 读取数量，再循环读取结构体。写和读的顺序必须一致。

### Q4：`qsort` 的比较函数为什么要返回差值？

参考回答：

`qsort` 通过比较函数判断两个元素的顺序。这里返回后一个元素的 `count` 减前一个元素的 `count`，可以实现按借用次数从大到小排序。

### Q5：为什么 `malloc` 后要判断是否为 `NULL`？

参考回答：

如果内存申请失败，`malloc` 会返回 `NULL`。如果不检查就继续使用，会导致程序崩溃。所以排行榜模块申请内存后先判断是否成功。

## 5. 故障分析题

### Q1：如果添加工具后退出，再打开数据没了，可能哪里有问题？

参考回答：

可能是退出时没有调用 `save_tools`，或者保存文件路径不对，也可能是文件打开失败。应该检查 `src/menu.c` 退出分支和 `src/storage.c` 的 `save_tools`。

### Q2：如果借用后库存没有减少，应该检查哪里？

参考回答：

检查 `borrow_tool` 中借用成功后的两行库存更新代码：

```c
tools[tool_idx].stock -= quantity;
tools[tool_idx].borrowed_count += quantity;
```

还要确认传入的是主程序里的同一个 `tools` 数组。

### Q3：如果归还后库存变多了很多，可能是什么问题？

参考回答：

可能是重复归还没有被拦截，应该检查 `return_tool` 中是否判断了记录状态为“已还”。如果没有这个判断，同一条记录可以反复归还，库存就会错误增加。

### Q4：如果程序查询不到明明存在的工具，可能是什么原因？

参考回答：

可能是查询方式选错了。编号查询是精准匹配，必须完全一致；名称和类型才是模糊查询。也可能是输入中包含多余字符，需要检查输入读取逻辑。

## 6. 改进题

### Q1：如果数据量超过 200 个工具怎么办？

参考回答：

当前用固定数组，最多 `MAX_TOOLS` 个。改进方案是使用链表或动态数组，根据实际数据量扩容。

### Q2：如果要支持按日期排序借还记录，怎么做？

参考回答：

可以把日期字符串按 `YYYY-MM-DD` 格式保存，这种格式可以直接按字符串比较；然后用排序函数对借还记录数组按日期字段排序。

### Q3：如果要导出 Excel，怎么改？

参考回答：

可以增加 CSV 导出功能，把工具信息和借还记录按逗号分隔写入 `.csv` 文件，Excel 可以直接打开。

### Q4：如果要做权限管理，怎么改？

参考回答：

可以增加用户结构体，保存用户名、密码和角色。管理员可以增删改工具，普通用户只能查询和借还。

### Q5：如果要避免二进制文件跨平台问题，怎么改？

参考回答：

可以改为文本格式或 CSV 格式保存。二进制直接写结构体依赖结构体布局，跨平台或字段变化时兼容性不如文本格式。
